CCIE Enterprise Core (ENCOR 350-401) Exam Topics

1. Architecture (15%)

1.1 Enterprise Network Design Principles

  • Hierarchical Network Design: Core, distribution, and access layers.
  • Modular Network Design: Campus, data center, WAN, and branch components. Collapsed core and spine-leaf architectures.
  • Software-Defined Networking (SDN): Cisco DNA Center for network automation and assurance. Overlay and underlay networks, including VXLAN.
  • Cloud Integrations: Hybrid and multi-cloud strategies. Integration of cloud-hosted services (AWS, Azure) with on-premises networks.

1.2 High Availability (HA) and Redundancy

  • HA Mechanisms: HSRP, VRRP, and GLBP for gateway redundancy. Redundant links with EtherChannel and link aggregation.
  • Load Balancing: Configuring load balancers for server farms. Traffic distribution techniques (round-robin, least connections).

1.3 Infrastructure Services

  • QoS (Quality of Service): Classification, marking (DSCP/CoS), shaping, and policing. QoS trust boundaries in wired and wireless networks.
  • Multicast: IGMP, PIM Sparse Mode (PIM-SM), and PIM Dense Mode (PIM-DM). Multicast group membership and traffic forwarding.

1.4 Network Automation and Programmability

  • Intent-Based Networking: How controllers (Cisco DNA Center) translate business intent into network configurations.
  • Model-Driven Telemetry: Real-time data collection using streaming telemetry.
  • API Integration: Integrating programmable solutions via REST APIs and NETCONF/RESTCONF.

2. Virtualization (10%)

2.1 Network Virtualization

  • GRE and IPsec Tunnels: Encapsulation techniques for secure data transport. Configuring site-to-site VPNs.
  • VXLAN: Extending Layer 2 across Layer 3 boundaries. VXLAN BGP EVPN for multi-tenancy.
  • VRFs: Virtual Routing and Forwarding for logical segmentation. Configuring route targets and leaking routes between VRFs.

2.2 Virtualization Technologies

  • SD-WAN: Cisco SD-WAN components (vEdge, vSmart, vBond). Overlay management protocols (OMP).
  • Virtualized Network Functions (VNFs): Deploying firewalls, routers, and load balancers as software instances.

3. Infrastructure (30%)

3.1 Layer 2 Technologies

  • VLANs and Trunking: Configuring 802.1Q trunks and VLAN tagging. Private VLANs and VLAN pruning.
  • Spanning Tree Protocol (STP): RSTP, MSTP, and troubleshooting loops. BPDU guard, root guard, and loop guard.
  • EtherChannel: Configuring LACP and PAgP for link aggregation.

3.2 Layer 3 Technologies

  • Routing Protocols: OSPFv2/v3, EIGRP, and BGP.
  • Policy-Based Routing (PBR): Redirecting traffic using route maps.
  • Multicast Routing: PIM-SM and Auto-RP configuration.

3.3 Wireless

  • Standards and Protocols: Differences between 802.11a/b/g/n/ac/ax.
  • WLAN Controllers: Configuring mobility groups and anchors. RF management and troubleshooting interference.

3.4 Infrastructure Services

  • DNS and DHCP: Configuring DHCP relay and troubleshooting leases.
  • NTP and SNMP: Time synchronization for devices. Monitoring via SNMPv2/v3.

4. Network Assurance (10%)

4.1 Monitoring Tools

  • Syslog and SNMP: Logging levels and SNMP traps.
  • Cisco DNA Assurance: Using AI/ML to identify and resolve network issues.
  • Telemetry: Push vs. pull models for data collection.

4.2 Troubleshooting

  • Root Cause Analysis: Analyzing packet flows with Wireshark. Troubleshooting link failures and protocol mismatches.

5. Security (20%)

5.1 Secure Network Access

  • 802.1X and MAB: Configuring port-based authentication.
  • Network Access Control (NAC): Cisco Identity Services Engine (ISE) policies for access.

5.2 Device Hardening

  • Control Plane Protection (CoPP): Protecting routers from DoS attacks.
  • Secure Management: Enforcing HTTPS, SSH, and SNMPv3.

5.3 VPNs

  • IPsec: Configuring site-to-site and DMVPN.
  • MACsec: Encrypting traffic on Layer 2.

6. Automation (15%)

6.1 Programming and APIs

  • Python Basics: Writing scripts for network device management.
  • APIs: Interacting with network devices using REST APIs.

6.2 Configuration Management

  • Ansible: Automating network configurations.
  • Git: Version control for Infrastructure as Code (IaC).

6.3 Model-Driven Programmability

  • YANG Models: Describing network configurations.
  • NETCONF/RESTCONF: Programmatically interacting with devices.
